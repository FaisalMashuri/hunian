Faisal adalah solo builder (kecil/indie) yang sedang membangun Hunian MVP.

**Konteks kerja:**
- Tim: solo / sangat kecil
- Runway: terbatas — kecepatan ship sangat penting
- Stage: validasi product-market fit, belum ada paying users
- Pendekatan yang cocok: small bets, ship cepat, validate, iterate

**Implikasi untuk PM:**
- Jangan rekomendasikan scope yang butuh lebih dari 3 minggu per slice
- Setiap fitur baru harus lolos: "apakah ini blocking completion rate, atau enrichment?"
- Selalu tanyakan dependency — AI prompt, integrations, dan state machine adalah sumber risiko terbesar untuk solo builder
- Validasi dengan 5 user nyata sebelum expand scope
